# Web Product Spec

## User

Primary user: admin or agent working from a laptop or desktop.

## MVP pages

### Auth

- login
- register

### Dashboard

- metric cards
- upcoming tasks
- recent clients
- pipeline summary

### Properties

- searchable table
- add and edit property form
- status badge

### Clients

- searchable table
- quick notes
- assigned agent
- interest type

### Deals

- kanban columns:
  - new
  - contacted
  - visit
  - negotiation
  - closed
- each card shows client, property, assigned agent, next task

### Tasks

- today
- overdue
- completed

## UX rules

- keep forms short
- use drawer or modal create forms if desired
- tables first, advanced analytics later
- pipeline must feel like the core value of the product

## Non-MVP items to skip

- accounting
- commission tracking
- full marketing automation
- map-heavy search
- document e-signature
