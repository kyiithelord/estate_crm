# Mobile Product Spec

## User

Primary user: field agent working on a phone while moving between calls, meetings, and property visits.

## Mobile MVP screens

### Sign in

- email
- password

### Home

- today's tasks
- deals needing follow-up
- quick actions

### Clients

- lead list
- tap to call
- tap to message
- quick note entry

### Deals

- grouped by stage
- stage update action sheet
- client and property summary

### Tasks

- due today
- overdue
- mark complete

### Properties

- simple searchable list
- detail page for client conversations

## Mobile UX rules

- prioritize speed over density
- keep one-thumb friendly actions
- use bottom tabs
- avoid large complex forms in MVP
- make reminders visible immediately on app open

## Mobile roadmap after MVP

- push notifications
- offline drafts
- camera upload for property photos
- voice notes
